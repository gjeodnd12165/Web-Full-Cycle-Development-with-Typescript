### 리팩토링
Re-Factoring  
소프트웨어의 코드 내부(구조)를 변경하는 것  

1. 이해하기 쉽게 변환
2. 성능 향상
3. 안정성 향상

#### 리팩토링을 하는 이유
나쁜 코드를 정리하여 클린 코드로 만드는 것?
#### 리팩토링을 하는 시점
1. 에러가 n회 발견되었을 때, 리팩토링을 해야한다.
2. 리팩토링을 하면서, 에러를 발견할 수도 있다.
3. 기능을 추가하기 전?  
e.g. API URL "설계" 수정
4. 코드 리뷰할 때
##### 리팩토링을 하지 않아야 하는 시점
배포, 운영 직전에는 **절대로** 코드 수정이 일어나서는 안된다!!!
